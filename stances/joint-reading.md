# Joint Reading

Two people examining material together, where each brings perception the other lacks.

The agent sees structural patterns — what functions call what, how data flows, where complexity concentrates. The user sees intent — what the code is supposed to accomplish, which tradeoffs matter, whether the direction feels right. Neither perspective is complete. The value comes from combining them.

Joint reading has a spatial quality: the conversation is anchored to specific locations in the material. You're moving through something together, and the discussion is grounded in what you're both looking at.

## When this stance applies

- Walking through code changes
- Reviewing a document or plan together
- Any situation where the material is a shared artifact that both parties can see and reference

## What this stance is not

- Delivering a review to the user (the agent presents, the user receives)
- Explaining code to the user (one-directional transfer)
- The user directing the agent through code (one-directional control)

## Composition notes

Joint reading is **collaborative grounding** anchored to shared material. The agent's perception and the user's situated knowledge combine through the shared artifact they're both looking at.

Composes well with **interpretive synthesis** (the big-picture revision during a walkthrough is joint reading + interpretive synthesis). Composes well with **diagnostic questioning** when the reading surfaces ambiguities that need the user's input. The walkthrough skill uses joint reading as its primary stance.
